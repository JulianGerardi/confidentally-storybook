# Vitals — formulario de carga

- **Archivo Figma:** UX-UI · 2.0
- **Página:** ↪ Vitals (node `877:17993`)
- **Node IDs:** `909:10995` (Default) · `1151:36273` (Default, duplicado) · `1151:37194` (Default, con `AlertMessage`) · `1151:35920` (Alerts)
- **URL:** https://www.figma.com/design/aESBq2SDwG2b4fnTyfBUgn/UX-UI-·-2.0?node-id=909-10995
- **Tamaño:** ~1064×1244–1355

## Descripción

Formulario clínico para registrar signos vitales del paciente, organizado en tarjetas por tipo de medición, con validación de rangos.

## Secciones principales

- Header + `ClinicalModeButton`
- Tarjetas de medición (`Card Title` + `Input` + texto de rango), incluye al menos:
  - Peso ("Weight measurement", en kg, "Range: 0 - 900 lbs")
  - Presión arterial ("Systolic / Diastolic measurement", "Range: Systolic 50-250 | Diastolic 30-150 mmHg")
  - Temperatura ("Measurement", "Range: 93 - 108 °F")
- 12× `Tabs / Trigger` (navegación entre categorías de vitales)
- 7× `Input`, 15× `Save Button` (guardado por tarjeta/medición)
- 5× `Button` (acciones generales)

## Variante: Vitals - Alerts (node `1151:35920`)

Misma estructura, agrega 4× `Badge` — indicadores de alerta cuando una medición está fuera de rango.

## Notas

- `909:10995` y `1151:36273` tienen la misma composición — probablemente el mismo estado duplicado.
- `1151:37194` agrega una instancia de `AlertMessage` (mensaje de validación a nivel de formulario).
